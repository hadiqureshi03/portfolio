Læring fra 02 / WEB

Introduktion til HTML & CSS

I dette første forløb, har jeg lært de fundamentale principper for opbygning af websider ved hjælp af HTML og CSS. Jeg startede med at forstå strukturen af HTML-dokumenter og hvordan man bruger semantiske tags til at organisere indhold på en meningsfuld måde.

En af de vigtigste læringer var forståelsen af responsivt design, og hvordan man kan skabe websites der fungerer godt på tværs af forskellige enheder og skærmstørrelser. Dette var helt centralt i Mobile Site projektet, hvor formålet var at skabe en mobilvenlig oplevelse.

Arbejde med CSS-layout

En stor del af projektet handlede om at lære at bruge CSS til at skabe layouts. Jeg blev særligt fortrolig med Flexbox og Grid, som er kraftfulde værktøjer til at organisere elementer på en webside.

Jeg lærte, at CSS Grid er særligt godt til todimensionelle layouts, hvor både rækker og kolonner er vigtige, mens Flexbox ofte er bedre til endimensionelle layouts, som f.eks. navigation eller en række kort.

.grid_1-1 { display: flex; } .grid_5-3 { display: grid; grid-template-columns: 3fr 1fr; } .grid_2-3 { display: grid; grid-template-columns: 1fr 2fr; gap: 2rem; justify-content: center; align-items: center; }

Ovenstående kodestykke viser, hvordan jeg har anvendt både Grid og Flexbox til forskellige layoutopgaver i projektet. Jeg brugte for eksempel forskellige grid-templates til at skabe varierede sektioner på siden. For navigation og kortvisninger anvendte jeg Flexbox (.grid_1-1), mens jeg brugte CSS Grid til mere komplekse layouts som indholdsområder med sidebjælker (.grid_5-3 med fordeling 3:1 og .grid_2-3 med fordeling 1:2).

Med properties som gap, justify-content og align-items lærte jeg at styre afstande og justere indhold præcist. Bemærk hvordan grid-template-columns definerer søjlernes proportioner ved hjælp af fr-enheden (fraction unit), hvilket gør layoutet fleksibelt og responsivt.

Brug af semantisk HTML

En anden vigtig lærdom var betydningen af semantisk HTML. Ved at bruge tags som <header>, <nav>, <main>, <section> og <footer> i stedet for generiske <div> tags, blev min kode mere meningsfuld og tilgængelig for skærmlæsere og søgemaskiner.

Dette har også gjort min kode mere vedligeholdelsesvenlig, da strukturen er tydeligere og lettere at forstå for andre udviklere (og mig selv efter noget tid).

Optimering af billeder

I projektet lærte jeg også vigtigheden af at optimere billeder for web. Ved at bruge moderne formater som WebP og korrekte størrelser, kunne jeg reducere sidens indlæsningstid markant, hvilket er særligt vigtigt for mobilbrugere, som måske har langsommere internetforbindelser.

Konklusion

Mobile Site projektet gav mig et stærkt fundament inden for webdesign og -udvikling. De færdigheder, jeg har tilegnet mig her, danner grundlag for alle mine fremtidige projekter, hvor jeg kan bygge videre på denne viden med mere avancerede teknikker og værktøjer.
